4信任链 · Auth谁颁发 · 谁验签

Web 用户视角

浏览器登录 server → session token (旧机制)

⇒ web 调 server: token header (业务路由 + 取 JWT)
⇒ web 调 server: POST /v1/auth/token (session) → JWT (web · 30min)
⇒ web 调 matter: Bearer JWT
            ↳ matter 用 server JWKS 公钥本地验签
⇒ web 调 fleet : Bearer JWT
            ↳ fleet 同上

JWT 在内存缓存 60s 余量自动 refresh
APIClient.ts interceptor 按 URL 模式自动注入

Daemon 视角

daemon 启动 → 持 api_key
                  │
daemon → server: POST /v1/auth/token (api_key)
                 ↳ 返回 JWT (daemon · 30 天)
                  │
⇒ daemon 调 fleet : Bearer JWT
            ↳ fleet 用 server JWKS 本地验签 ✓
⇒ daemon 调 matter: Bearer JWT
            ↳ matter 同上 ✓
⇒ daemon 调 server: GET /v1/bot/:uid/token (Bearer JWT)
            ↳ server 验签 + 校验 robot.creator_uid==JWT.sub
            ↳ 返回 bot_token 喂 openclaw

unknown kid 自动触发 JWKS refresh (10s floor)
server 是唯一 JWT issuer · RS256 · kid 跨重启稳定（hash of modulus）· fleet/matter 启动时拉一次 /.well-known/jwks.json 缓存。零互调，3 服务之间不发 HTTP。
